### Краткое описание проекта
Это Real-time мессенджер, построенный на базе FastAPI и Socket.IO. Redis используется в качестве брокера для Celery. Есть имитация обработки в задаче celery.

### Текущий технологический стек

- Backend: FastAPI (для HTTP-запросов и API).
- Real-time: Python-SocketIO (для мгновенного обмена сообщениями).
- Message Broker: Redis (служит «мостом» между сервером сокетов и фоновыми задачами).
- Task Queue: Celery (для обработки тяжелых операций вне основного потока приложения).
- Server: ASGI (Uvicorn) для асинхронной работы.

### Архитектура и слои
- Core (Ядро): Центр управления. Здесь инициализируется сервер Socket.IO с подключенным Redis-менеджером и настраивается конфиг Celery.
- Sockets (Транспорт): Слой обработки событий реального времени. Здесь описана логика подключения пользователей, их входа в комнаты и отправки сообщений.
- API (Маршруты): Стандартные HTTP-эндпоинты (например, проверка статуса сервиса).
- Tasks (Фоновые задачи): Слой «тяжелой» логики. Когда приходит сообщение, сокет мгновенно пересылает его клиенту, а Celery-воркер параллельно выполняет обработку данных (сохранение логов, модерацию и т.д.).
- Main (Точка входа): Сборка всех компонентов в единое ASGI-приложение.


### Запуск 
- 1 терминал: перейти в папку, где находится Redis (прим. C:/Redis), прописать: redis-server
- 2 терминал: перейти в папку проекта и прописать python -m celery -A core.celery_app worker --loglevel=info, в папках core, sockets, tasks желательно добавить __ init __ файлы.
- 3 терминал: перейти в папку проекта и прописать uvicorn main:app --reload 
--host 0.0.0.0 --port 8000
